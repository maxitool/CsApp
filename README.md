<h3>1.
Задание:</h3>
Первый метод должен принимать на вход csv, после чего происходит обработка и сохранение данных файла в БД.
CSV имеет следующий формат:
Date;ExecutionTime;Value
<Время начала ГГГГ-ММ-ДДTчч-мм-сс.ммммZ>;<Время выполнения в секундах>;<Показатель в виде числа с плавающей запятой>
Файл парсится, значения записываются в базу в таблицу Values. Необходимо выполнить валидацию:
1) Дата не может быть позже текущей и раньше 01.01.2000
2) Время выполнения не может быть меньше 0
3) Значение показателя не может быть меньше 0
4) Количество строк не может быть меньше 1 и больше 10 000
5) Значения должны соответствовать своим типам, отсутствие одного из значений в записи недопустимо.
Если какое-либо условие не выполнено, нужно считать файл невалидным, откатить изменения и вернуть пользователю соовтетствующую ошибку.
Также из значений файла подсчитываются интегральные результаты и записываются в таблицу Results:
1) Дельта времени Date в секундах (максимальное Date – минимальное Date)
2) Минимальное дата и время, как момент запуска первой операции (Date)
3) Среднее время выполнения (ExecutionTime)
4) Среднее значение по показателям (Value)
5) Медина по показателям (Value)
6) Максимальное значение показателя (Value)
7)Минимальное значение показателя (Value)
Если файл с таким именем уже существует, необходимо перезаписывать значения в базе.

Реализовано в методе TimescaleDataProcessing/SaveToDB
В аргументе данный метод принимает файл. 
Вывод либо сообщение о успехе, либо о провале валидации.

<h3>2.
Задание:</h3>
Второй метод предназначен для получения списка записей из таблицы Results, подходящих под фильтры.
Могут применяться фильтры:
1) По имени файла
2) По времени запуска первой операции (диапазон)
3) По среднему показателю (диапазон)
5) По среднему времени выполнению (диапазон)

Реализовано в методе TimescaleDataProcessing/GetResultsWithFilters
В аргументе данный метод принимает класс с полями.
Пример ввода данных:
{
  "haveFilenameFilter": true,
  "filename": "TimescaleData.csv",
  "haveMinDateFilter": true,
  "fromDate": "2024-07-15T14:30:45.1234Z",
  "toDate": "2026-07-15T14:30:45.1234Z",
  "haveAvgValueFilter": true,
  "fromAvgValue": "234.1",
  "toAvgValue": "234.2",
  "haveAvgExecutionFilter": true,
  "fromAvgExecution": "213123123.0",
  "toAvgExecution": "213123123.1"
}
где haveFilenameFilter - будет ли использовать фильтр по имени файла (filename)
haveMinDateFilter - будет ли использовать фильтр По времени запуска первой операции (диапазон (fromDate, toDate))
haveAvgValueFilter - будет ли использовать фильтр по среднему показателю (диапазон (fromAvgValue, toAvgValue))
haveAvgExecutionFilter - будет ли использовать фильтр по среднему времени выполнению (диапазон (fromAvgExecution, toAvgExecution))

Обратине внимание на формат дат, они в виде ГГГГ-ММ-ДДTчч-мм-сс.ммммZ, как было заявлено в задании.
Возможные выводы:
1) Отсортированные по фильтрам данные с таблицы Results
2) Все данные с таблицы Results, если не были указаны фильтры.
3) Ошибка о провале валидации.
