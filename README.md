<h2>Реализовано:</h2>
<h3>1.
Задание:</h3>
<p>Первый метод должен принимать на вход csv, после чего происходит обработка и сохранение данных файла в БД.</p>
CSV имеет следующий формат:
<p>Date;ExecutionTime;Value</p>
<Время начала ГГГГ-ММ-ДДTчч-мм-сс.ммммZ>;<Время выполнения в секундах>;<Показатель в виде числа с плавающей запятой>
<p>Файл парсится, значения записываются в базу в таблицу Values. Необходимо выполнить валидацию:</p>
<p>1) Дата не может быть позже текущей и раньше 01.01.2000</p>
<p>2) Время выполнения не может быть меньше 0</p>
<p>3) Значение показателя не может быть меньше 0</p>
<p>4) Количество строк не может быть меньше 1 и больше 10 000</p>
<p>5) Значения должны соответствовать своим типам, отсутствие одного из значений в записи недопустимо.</p>
<p>Если какое-либо условие не выполнено, нужно считать файл невалидным, откатить изменения и вернуть пользователю соовтетствующую ошибку.</p>
<p>Также из значений файла подсчитываются интегральные результаты и записываются в таблицу Results:</p>
<p>1) Дельта времени Date в секундах (максимальное Date – минимальное Date)</p>
<p>2) Минимальное дата и время, как момент запуска первой операции (Date)</p>
<p>3) Среднее время выполнения (ExecutionTime)</p>
<p>4) Среднее значение по показателям (Value)</p>
<p>5) Медина по показателям (Value)</p>
<p>6) Максимальное значение показателя (Value)</p>
<p>7)Минимальное значение показателя (Value)</p>
<p>Если файл с таким именем уже существует, необходимо перезаписывать значения в базе.</p>

</br>
<p><b>Реализовано</b> в методе TimescaleDataProcessing/SaveToDB</p>
<p>В аргументе данный метод принимает файл. </p>
<p>Вывод либо сообщение о успехе, либо о провале валидации.</p>

<h3>2.
Задание:</h3>
<p>Второй метод предназначен для получения списка записей из таблицы Results, подходящих под фильтры.
Могут применяться фильтры:</p>
<p>1) По имени файла</p>
<p>2) По времени запуска первой операции (диапазон)</p>
<p>3) По среднему показателю (диапазон)</p>
<p>5) По среднему времени выполнению (диапазон)</p>

</br>
<p><b>Реализовано</b> в методе TimescaleDataProcessing/GetResultsWithFilter В виде Chain of Responsibility pattern.</p>
<p>В аргументе данный метод принимает класс с полями.</p>
<p>Пример ввода данных:</p>
<p>{</p>
<p>  "haveFilenameFilter": true,</p>
<p>  "filename": "TimescaleData.csv",</p>
<p>  "haveMinDateFilter": true,</p>
<p>  "fromDate": "2024-07-15T14:30:45.1234Z",</p>
<p>  "toDate": "2026-07-15T14:30:45.1234Z",</p>
<p>  "haveAvgValueFilter": true,</p>
<p>  "fromAvgValue": "234.1",</p>
<p>  "toAvgValue": "234.2",</p>
<p>  "haveAvgExecutionFilter": true,</p>
<p>  "fromAvgExecution": "213123123.0",</p>
<p>  "toAvgExecution": "213123123.1"</p>
<p>}</p>
<p>где haveFilenameFilter - будет ли использовать фильтр по имени файла (filename)</p>
<p>haveMinDateFilter - будет ли использовать фильтр По времени запуска первой операции (диапазон (fromDate, toDate))</p>
<p>haveAvgValueFilter - будет ли использовать фильтр по среднему показателю (диапазон (fromAvgValue, toAvgValue))</p>
<p>haveAvgExecutionFilter - будет ли использовать фильтр по среднему времени выполнению (диапазон (fromAvgExecution, toAvgExecution))</p>

<p>Обратине внимание на формат дат, они в виде ГГГГ-ММ-ДДTчч-мм-сс.ммммZ, как было заявлено в задании.
Возможные выводы:</p>
<p>1) Отсортированные по фильтрам данные с таблицы Results</p>
<p>2) Все данные с таблицы Results, если не были указаны фильтры.</p>
<p>3) Ошибка о провале валидации.</p>
